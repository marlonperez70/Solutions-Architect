# Enterprise Platform - Arquitectura Completa\n\n## Visión General\n\nLa Plataforma Empresarial es un sistema de misión crítica diseñado para operar en entornos corporativos de alta exigencia (banca, fintech, retail, telecomunicaciones). Implementa arquitectura de microservicios, event-driven architecture, Domain-Driven Design (DDD), Clean Architecture, Cloud Native, API First Design y Zero Trust Security.\n\n## Capas de Arquitectura\n\n### 1. Zero Trust\n\n### Principios\n1. **Never Trust, Always Verify**: Cada solicitud se valida independientemente\n2. **Least Privilege**: Acceso mínimo necesario\n3. **Context-Based Access**: Decisiones basadas en contexto (IP, User Agent, tiempo)\n4. **Continuous Validation**: Validación constante de identidad y permisos\n\n### Implementación\n\n#### Risk Scoring\n```typescript\nfunction calculateRiskScore(ctx, action): number {\n  let score = 0;\n  \n  // Factor 1: IP changes (0-20 points)\n  if (ctx.user.lastIpAddress !== ctx.req.ip) {\n    score += 20;\n  }\n  \n  // Factor 2: User Agent changes (0-15 points)\n  if (ctx.user.lastUserAgent !== ctx.req.headers['user-agent']) {\n    score += 15;\n  }\n  \n  // Factor 3: Time anomalies (0-15 points)\n  const timeSinceLastSignIn = now - ctx.user.lastSignedIn;\n  if (timeSinceLastSignIn > 7 days) {\n    score += 15;\n  }\n  \n  // Factor 4: Sensitive operations (0-25 points)\n  if (isSensitiveAction(action)) {\n    score += 25;\n  }\n  \n  return Math.min(score, 100);\n}\n```\n\n#### Validación de Contexto\n```typescript\nasync function validateSecurityContext(\n  ctx: TrpcContext,\n  action: string,\n  requiredRole?: Role\n): Promise<ValidationResult> {\n  // 1. Autenticación\n  if (!ctx.user) return { isValid: false, riskScore: 100 };\n  \n  // 2. Estado activo\n  if (!ctx.user.isActive) return { isValid: false, riskScore: 100 };\n  \n  // 3. RBAC\n  if (requiredRole && ctx.user.role !== requiredRole) {\n    return { isValid: false, riskScore: 90 };\n  }\n  \n  // 4. Risk scoring\n  const riskScore = await calculateRiskScore(ctx, action);\n  \n  // 5. Logging\n  await createAuditLog({\n    userId: ctx.user.id,\n    action,\n    riskScore,\n    ipAddress: ctx.req.ip,\n    userAgent: ctx.req.headers['user-agent'],\n  });\n  \n  // 6. Decision\n  return {\n    isValid: riskScore < 80,\n    riskScore,\n  };\n}\n```\n\n#### Rate Limiting\n```typescript\nclass RateLimiter {\n  private attempts: Map<string, number[]> = new Map();\n  \n  isAllowed(key: string, maxAttempts: number, windowMs: number): boolean {\n    const now = Date.now();\n    const attempts = this.attempts.get(key) || [];\n    \n    const recentAttempts = attempts.filter(\n      (time) => now - time < windowMs\n    );\n    \n    if (recentAttempts.length >= maxAttempts) {\n      return false;\n    }\n    \n    recentAttempts.push(now);\n    this.attempts.set(key, recentAttempts);\n    return true;\n  }\n}\n\n// Limitadores globales\nconst loginRateLimiter = new RateLimiter();     // 5 intentos / 15 min\nconst apiRateLimiter = new RateLimiter();       // 100 req / min\nconst sensitiveActionLimiter = new RateLimiter(); // 10 acciones / hora\n```\n\n## Gestión de Agentes de IA\n\n### Tipos de Agentes\n\n1. **Data Analyst

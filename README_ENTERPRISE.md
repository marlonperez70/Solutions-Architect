# Enterprise Platform - Plataforma Empresarial de Misión Crítica\n\n## Descripción\n\nPlataforma empresarial robusta y escalable diseñada para operar en entornos corporativos de alta exigencia (banca, fintech, retail, telecomunicaciones). Implementa arquitectura de microservicios, event-driven architecture, Domain-Driven Design (DDD), Clean Architecture, Cloud Native, API First Design y Zero Trust Security.\n\n## Características Principales\n\n### 1. Autenticación y Autorización\n- ✅ Autenticación OAuth integrada con Manus\n- ✅ Gestión de roles (admin, user, agent)\n- ✅ Zero Trust Security Model\n- ✅ Risk scoring basado en contexto\n- ✅ Rate limiting por operación\n- ✅ Auditoría completa de acciones\n\n### 2. Gestión de Agentes de IA\n- ✅ 5 tipos de agentes especializados\n  - Data Analyst: Análisis de datos\n  - Security Auditor: Auditoría de seguridad\n  - Performance Monitor: Monitoreo de rendimiento\n  - Business Intelligence: Inteligencia de negocio\n  - Custom: Agentes personalizados\n- ✅ Integración con LLM\n- ✅ Integración MCP para acceso a BD\n- ✅ Ejecución y tracking de agentes\n\n### 3. Monitoreo y Observabilidad\n- ✅ Event Sourcing para auditoría completa\n- ✅ Logging estructurado\n- ✅ Almacenamiento en S3\n- ✅ Métricas de negocio y técnicas\n- ✅ Dashboards en tiempo real\n\n### 4. Sistema de Alertas\n- ✅ 6 tipos de alertas\n- ✅ 4 niveles de severidad\n- ✅ Notificaciones por email\n- ✅ Alertas automáticas por eventos críticos\n- ✅ Digest diario\n\n### 5. Generación de Gráficos con IA\n- ✅ Generación automática de visualizaciones\n- ✅ Dashboards ejecutivos\n- ✅ Reportes inteligentes\n- ✅ Análisis de tendencias\n\n### 6. Almacenamiento Empresarial\n- ✅ Almacenamiento en S3\n- ✅ Versionado de documentos\n- ✅ Gestión de permisos\n- ✅ Auditoría de acceso\n\n## Arquitectura Técnica\n\n### Stack Tecnológico\n\n**Frontend**:\n- React 19\n- TypeScript\n- Next.js\n- Tailwind CSS 4\n- tRPC Client\n\n**Backend**:\n- Express 4\n- tRPC 11\n- Node.js 22\n- Drizzle ORM\n\n**Base de Datos**:\n- MySQL/TiDB\n- 8 tablas principales\n- Índices optimizados\n- Relaciones definidas\n\n**Servicios Externos**:\n- Manus OAuth\n- Manus LLM\n- Manus S3\n- SMTP para emails\n\n### Tablas de Base de Datos\n\n1. **users** - Gestión de usuarios\n2. **agents** - Agentes especializados\n3. **events** - Event Sourcing\n4. **alerts** - Sistema de alertas\n5. **auditLogs** - Auditoría de acciones\n6. **documents** - Almacenamiento de archivos\n7. **metrics** - Métricas empresariales\n8. **configurations** - Configuración por entorno\n\n## Instalación\n\n### Requisitos Previos\n- Node.js 22+\n- pnpm 10+\n- MySQL/TiDB\n- Git\n\n### Pasos de Instalación\n\n```bash\n# Clonar repositorio\ngit clone https://github.com/tu-usuario/enterprise-platform.git\ncd enterprise-platform\n\n# Instalar dependencias\npnpm install\n\n# Configurar variables de entorno\ncp .env.example .env\n\n# Ejecutar migraciones\npnpm drizzle-kit generate\npnpm drizzle-kit migrate\n\n# Iniciar servidor de desarrollo\npnpm dev\n```\n\n## Configuración\n\n### Variables de Entorno\n\n```bash\n# Database\nDATABASE_URL=mysql://user:password@localhost:3306/enterprise_platform\n\n# OAuth\nVITE_APP_ID=your-app-id\nOAUTH_SERVER_URL=https://api.manus.im\nVITE_OAUTH_PORTAL_URL=https://oauth.manus.im\n\n# Email\nSMTP_HOST=smtp.gmail.com\nSMTP_PORT=587\nSMTP_USER=your-email@gmail.com\nSMTP_PASSWORD=your-app-password\nSMTP_FROM=noreply@enterprise-platform.local\n\n# S3\nAWS_ACCESS_KEY_ID=your-key\nAWS_SECRET_ACCESS_KEY=your-secret\nAWS_REGION=us-east-1\nAWS_S3_BUCKET=your-bucket\n\n# LLM\nOPENAI_API_KEY=your-key\n```\n\n## Uso\n\n### Desarrollo\n\n```bash\n# Iniciar servidor de desarrollo\npnpm dev\n\n# Ejecutar tests\npnpm test\n\n# Verificar tipos\npnpm check\n\n# Formatear código\npnpm format\n```\n\n### Producción\n\n```bash\n# Build\npnpm build\n\n# Start\npnpm start\n```\n\n## API Endpoints\n\n### Agentes\n- `agents.list()` - Listar agentes\n- `agents.getById(id)` - Obtener agente\n- `agents.create(data)` - Crear agente\n- `agents.update(id, data)` - Actualizar agente\n\n### Eventos\n- `events.recent(limit)` - Eventos recientes\n- `events.create(data)` - Crear evento\n\n### Alertas\n- `alerts.open(limit)` - Alertas abiertas\n- `alerts.create(data)` - Crear alerta\n- `alerts.updateStatus(id, status)` - Actualizar estado\n\n### Métricas\n- `metrics.byCategory(category)` - Métricas por categoría\n- `metrics.record(data)` - Registrar métrica\n\n### Usuarios\n- `users.list()` - Listar usuarios (admin only)\n\n### Auditoría\n- `audit.myLogs(limit)` - Mis logs de auditoría\n\n## Seguridad\n\n### Zero Trust Security\n\nCada solicitud se valida mediante:\n1. **Autenticación**: Usuario debe estar autenticado\n2. **Autorización**: Usuario debe tener rol requerido\n3. **Risk Scoring**: Cálculo de riesgo basado en:\n   - Cambios de IP\n   - Cambios de User Agent\n   - Anomalías de tiempo\n   - Sensibilidad de operación\n4. **Rate Limiting**: Límites por operación\n5. **Auditoría**: Logging de todas las acciones\n\n### Protecciones\n- ✅ Sanitización de entrada\n- ✅ Validación de email\n- ✅ Hashing de datos sensibles en logs\n- ✅ HTTPS obligatorio\n- ✅ CORS configurado\n- ✅ CSRF protection\n\n## Monitoreo\n\n### Logs Estructurados\n\nTodos los eventos se registran con:\n```json\n{\n  \"timestamp\": \"2026-03-13T06:00:00Z\",\n  \"level\": \"info\",\n  \"service\": \"agent-service\",\n  \"userId\": 1,\n  \"action\": \"execute_analysis\",\n  \"resource\": \"agent\",\n  \"status\": \"success\",\n  \"duration\": 1250,\n  \"metadata\": {}\n}\n```\n\n### Almacenamiento en S3\n- Logs archivados diariamente\n- Retención de 90 días\n- Compresión automática\n\n## Testing\n\n### Ejecutar Tests\n\n```bash\npnpm test\n```\n\n### Cobertura\n\n```bash\npnpm test -- --coverage\n```\n\n### Tests Disponibles\n- ✅ Autenticación\n- ✅ Autorización\n- ✅ Gestión de agentes\n- ✅ Eventos\n- ✅ Alertas\n- ✅ Seguridad\n\n## Despliegue\n\n### Docker\n\n```bash\n# Build image\ndocker build -t enterprise-platform:latest .\n\n# Run container\ndocker run -p 3000:3000 enterprise-platform:latest\n```\n\n### Kubernetes\n\n```bash\nkubectl apply -f k8s/\n```\n\n### CI/CD\n\nConfigurado con GitHub Actions:\n- Tests automáticos\n- Build automático\n- Deploy automático\n\n## Documentación\n\n- [ARCHITECTURE.md](./ARCHITECTURE.md) -
